## Supported Platforms

These templates are designed to work across the tools small businesses actually use:

### Google Sheets
- **Best for:** Teams under 25 people, businesses without a dedicated analyst
- **What you get:** Formulas, conditional formatting rules, named ranges, and data validation all specified in template docs and auto-generated by `sheets-kpi-tracker.py`
- **Limitations:** Performance degrades past ~50,000 rows; no real-time streaming

### Looker Studio (formerly Google Data Studio)
- **Best for:** Teams that need polished, shareable reports connected to live data
- **What you get:** Layout specs and metric definitions map directly to Looker Studio scorecards, time series charts, and filter controls
- **Data connections:** Google Sheets, BigQuery, Google Analytics, CSV upload, and 800+ community connectors

### Microsoft Excel / Power BI
- **Best for:** Organizations already invested in the Microsoft ecosystem
- **What you get:** Formulas use Excel-compatible syntax; layout specs translate to Power BI report pages
- **Note:** Some Google Sheets functions (SPARKLINE, GOOGLEFINANCE) have Power BI equivalents noted in templates

### Notion / Airtable
- **Best for:** Teams using these as their operational backbone
- **What you get:** KPI definitions and formulas can be implemented as Notion database properties or Airtable formula fields
- **Limitation:** Visualization capabilities are more limited; consider pairing with Looker Studio for charts

### Standalone HTML Reports
- **Best for:** One-off reports, client deliverables, email attachments
- **What you get:** The `csv-to-dashboard.py` script generates self-contained HTML files with no external dependencies
- **Advantage:** Works offline, can be emailed, no login required

## How to Use This Collection

### Step 1: Select Your KPIs

Start with [`frameworks/kpi-selection-guide.md`](frameworks/kpi-selection-guide.md). Find your industry vertical and pick 5-8 KPIs from the recommended list. Resist the urge to track everything. The most effective dashboards we have built track 6 metrics well rather than 20 metrics poorly.

### Step 2: Choose a Dashboard Template

Pick the template closest to your primary audience:

- **Executive team or board?** Start with the [Executive Summary Dashboard](templates/executive-summary-dashboard.md)
- **Sales team or revenue leader?** Start with the [Sales Pipeline Dashboard](templates/sales-pipeline-dashboard.md)
- **Operations or fulfillment team?** Start with the [Operations Health Dashboard](templates/operations-health-dashboard.md)

You will likely need more than one dashboard eventually. Start with one, get it working, and expand.

### Step 3: Review Design Principles

Before building, read [`frameworks/dashboard-design-principles.md`](frameworks/dashboard-design-principles.md). It will save you from the most common mistakes: cramming too much data onto one screen, using colors that confuse rather than clarify, and building dashboards that go stale because no one owns the update process.

### Step 4: Build It

**Option A — Google Sheets (fastest start):**
```bash
pip install google-auth google-auth-oauthlib google-api-python-client
python scripts/sheets-kpi-tracker.py
```
This creates a formatted tracking spreadsheet in your Google Drive. See the script comments for Google Cloud project setup.

**Option B — HTML Report from CSV:**
```bash
python scripts/csv-to-dashboard.py your_metrics.csv --output dashboard.html
```
Generates a standalone HTML dashboard you can open in any browser.

**Option C — Manual build in your preferred tool:**
Use the template specs as blueprints. Each template includes metric definitions, layout guidance, and formula references you can implement in any spreadsheet or BI tool.

### Step 5: Establish Cadence

A dashboard without a review cadence is a dashboard that dies. Set a recurring calendar event:

| Dashboard Type | Review Frequency | Attendees |
|----------------|-----------------|-----------|
| Executive Summary | Weekly (Monday AM) | Leadership team |
| Sales Pipeline | Daily standup + weekly deep dive | Sales team + manager |
| Operations Health | Daily check + weekly review | Ops team + leads |

## Common Mistakes We See

1. **Tracking too many KPIs.** If your dashboard has more than 10 metrics, you do not have a dashboard. You have a data dump. Prioritize ruthlessly.

2. **No single owner.** Every dashboard needs one person responsible for data accuracy and update cadence. Not a committee. One person.

3. **Vanity metrics.** Page views, social media followers, and total registered users feel good but rarely drive decisions. Focus on metrics that change behavior.

4. **No targets.** A number without a target is just a number. Every KPI should have a goal, a threshold for "acceptable," and a threshold for "investigate immediately."

5. **Stale data.** A dashboard showing last month's data in a weekly review meeting destroys trust in the entire system. Automate data refresh or assign a clear owner for manual updates.

6. **Building before defining.** Do not open Google Sheets first. Define what decisions the dashboard should inform, then work backward to the metrics that support those decisions.
