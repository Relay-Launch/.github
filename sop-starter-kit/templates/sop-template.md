## 5. Procedure

<!--
  This is the core of the SOP. Write the steps in the exact order they should be performed.

  Guidelines:
  - Number every step. Use sub-steps (5a, 5b) for branching within a step.
  - Start each step with a verb: "Open," "Enter," "Send," "Verify," "Review."
  - Be specific: include exact tool names, menu paths, file locations, and field names.
  - If a step produces an output or artifact, say what it is and where it goes.
  - If a step has a time constraint, state it: "within 24 hours," "by end of business day."
  - If a step requires a decision, describe the criteria and the paths clearly.
  - Add brief notes (indented, in italics) to explain WHY a step matters when the
