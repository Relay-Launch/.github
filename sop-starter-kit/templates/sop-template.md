## 5. Procedure

<!--
This is the core of the SOP. Write numbered steps in the order they are
performed. Guidelines:

- Be specific: include system names, menu paths, field names, URLs
- Note who performs each step if responsibility shifts during the process
- Include expected timeframes where relevant
- Call out decision points clearly (use sub-steps for branching logic)
- Add screenshots or links to reference materials where helpful
- If a step is complex, break it into sub-steps (5a, 5b, 5c)

Keep each step focused on a single action. If you are writing "and" in a
step, consider whether it should be two steps.
